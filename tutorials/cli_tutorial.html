ChimeraMapR  ·  v0.7.0

Running chimera_cli.R from the terminal

A hands-on walkthrough of ChimeraMapR's batch command-line interface — the same haplotype-switch analysis as the Shiny app, driven from a shell prompt against real yeast chromosome data.

R ≥ 4.1 Rscript chimera_cli.R S. cerevisiae chrI test data

What you need first

  • R ≥ 4.1, with shiny, data.table, ggplot2, pracma, Matrix, Rcpp, igraph, and optparse installed (optparse is only needed for the CLI).
  • A C++ toolchain (Xcode CLT on macOS) — chimera_functions.R compiles a Rcpp HMM the first time it's sourced.
  • Three input files: a read-level SNP call CSV, a SNP definition file (CSV or VCF), and a .fai chromosome size index. See Input files in the README for the exact schema — this tutorial uses ready-made copies from test_data/.
No install step needed to try this. If you've cloned the ChimeraMapR repo, test_data/chrI.csv.gz, test_data/chrI_SNPs.vcf.gz, and test_data/ref_genome.fasta.fai are already there — every command below runs as-is from the repo root.
01

Check the option list

The CLI takes three positional arguments — read data, SNP data, chromosome sizes — plus flags for filtering, tuning, and output mode. Run it with no arguments (or --help) to see the full list with defaults baked in:

terminal
$ Rscript chimera_cli.R --help

Usage: chimera_cli.R [options] <read_data> <snp_data> <chr_size.fai>

ChimeraMapR command-line interface.
Detects chimeric haplotypes in long-read sequencing data.

Options:
	-n NAME, --sample-name=NAME
		Sample name used in output filenames and plot titles [default: Sample_01]
	--mapq-cutoff=INT
		Minimum MAPQ value; reads below this are excluded [default: 20]
	--baseq-cutoff=INT
		Minimum base quality at SNP position [default: 10]
	--min-run=INT
		Minimum consecutive same-allele calls to count as a run [default: 2]
	--min-peak-height=INT
		Minimum transition count at a SNP to qualify as a peak [default: 10]
	-l FLOAT, --lambda=FLOAT
		Whittaker smoothing penalty lambda (lower = tighter fit) [default: 1]
	--peak-list
		Write a CSV of detected peaks instead of the overview plot
	--overview-rds
		Save the overview plot as an RDS object for re-plotting in R
	--coverage-map
		Also write per-position coverage + collapsed segments as CSVs
	--chain-all
		Run the chain-based LOH event caller (step 0–4 CSVs)
	-o PATH, --output=PATH
		Output file path or directory [auto-named if omitted]

Three output modes exist and are mutually exclusive: the default PNG overview, --peak-list, and --overview-rds. Everything else — coverage maps, chain analysis — layers on top of whichever one you pick.

02

Run the default overview plot

With no output flags, the CLI classifies every read, detects chimeric transitions, fits a Whittaker-smoothed curve per chromosome, and saves a genome-wide PNG. Point it at chromosome I of the bundled test set:

    --sample-name Tutorial_chrI \

The WARNING line is the del-rate filter working as intended — two SNP positions had too many local deletions to trust their base calls, so chimera_functions.R dropped them before allele classification rather than let noisy calls corrupt a peak. That is normal, not an error.

04

Call recombination events with the chain analysis

--chain-all runs the full LOH + peak-fusion + event-calling pipeline and writes one CSV per pass, so you can inspect every intermediate stage rather than trusting a black box:

terminal
$ Rscript chimera_cli.R --sample-name Tutorial_chrI --chain-all \
    --output results/chain/ \
    test_data/chrI.csv.gz test_data/chrI_SNPs.vcf.gz test_data/ref_genome.fasta.fai

The number climbs from 6 raw peaks to 13 tokens (step 1) because the chain builder also encodes the flat, non-chimeric stretches between peaks as their own tokens — then step 2 merges any that share state across a small gap. What you actually want is the last file:

The internal_crossover peak from step 03 resolved into a CO_GC_subres call — a gene-conversion event below the tract-resolution floor, flagged review because it has peak support but no independently resolvable LOH tract. That distinction (a real call with a confidence flag, vs. a peak sitting unclaimed) is the whole point of running the chain analysis instead of stopping at the peak list.

Requires loh_chain_analysis.R next to chimera_functions.R. The CLI checks for it at startup and errors immediately if --chain-all is passed without it, rather than failing partway through.
05

Tune the analysis

Every parameter exposed in the Shiny sidebar has a matching flag. The defaults are reasonable for typical long-read coverage — reach for these when a run looks too noisy or too conservative:

FlagWhat it does
--min-peak-heightRaise this if the overview plot is cluttered with low-support peaks; lower it for shallow-coverage samples where 10 reads never accumulate at a real breakpoint.
--min-runHow many consecutive same-allele calls make a "run." Raise it on noisier base calls to avoid spurious single-SNP switches.
-l, --lambdaWhittaker smoothing penalty. Lower values track sharp peaks more tightly; higher values smooth over jitter at the cost of resolution.
--mapq-cutoff / --baseq-cutoffRead and base-quality floors applied before any allele classification happens.
--tel-tol / --merge-gapChain-analysis knobs: how close to a chromosome end counts as terminal, and how large an NA gap between same-state segments still gets merged.
--min-spanMinimum spanning reads for a read-based chain call — below this, chain analysis reports the segment as ambiguous instead of guessing.

Change one parameter at a time and re-run with --peak-list first — it's the fastest way to see how a threshold shift changes what gets called a peak, before committing to a full --chain-all pass.

06

Where things land

Every mode respects --output the same way: a bare path is treated as the exact file to write, a directory (or a path ending in /) gets an auto-generated <sample>_chimera_<date> filename inside it, and any missing parent directories are created for you. --coverage-map and --chain-all write their extra CSVs into that same resolved directory, alongside the primary output.

<sample>_chimera_<date>.pngDefault mode — genome-wide overview plot
<sample>_chimera_<date>_peaks.csv--peak-list — every detected peak with haplotype label
<sample>_chimera_<date>_overview.rds--overview-rds — re-plot later with readRDS() + print()
<sample>_<date>_coverage_table.csv / _coverage_segments.csv--coverage-map — per-position depth and collapsed segments
<sample>_<date>_chain_step0…step4_*.csv--chain-all — one CSV per pass, ending in step4_final_events.csv
Full flag reference, input-file schemas, and the underlying analysis pipeline are documented in the repo README.md. This walkthrough used the chromosome I test set bundled at test_data/ — swap in your own bamCol.py-derived read CSV, VCF, and samtools faidx index to run it on real data.
